## 設計メモ（意思決定の記録）

### 2026-06-05: エンジン選定
Unity 2022.3 LTS を選択。
UniTask + DOTween の成熟度とScriptableObjectアーキテクチャとの親和性を優先。
Unreal案はGASの学習コストと2D/3Dハイブリッドの複雑さで却下。

### 2026-06-05: コアシステム「デュアルペルソナ」採用決定
主人公が2枚のペルソナを保持できる仕組み。
単なるペルソナシリーズの模倣ではなく「演出家（ディレクター）」というコアテーマを
「二枚使い = 同時に複数の視点を持てる」という形でシステム化した。
これがこのゲームの差別化ポイント①。

### 2026-06-05: 劇場プログラムUI方針確定
全てのメニュー・UIが「舞台プログラム / マーキー電光掲示板」のビジュアル言語を使う。
UIを開く行為そのものが「劇場を訪れる体験」に繋がるよう設計する。
これがこのゲームの差別化ポイント②。

### 2026-06-05: エンディング構造の方針
トゥルーエンドの条件を「全コープLv10 + 素顔力3項目MAX + 特定選択」にした。
これにより「全コープを育てること自体がストーリー」という構造が成立する。
バッドエンドは「罰」ではなく「もう一つの世界線」として描く（P3のバッドエンドを参考）。
