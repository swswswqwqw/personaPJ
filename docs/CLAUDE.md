## 今夜の作業サマリー

### 何をやったか
- **DESIGN.md（聖典）の完全作成**: タイトル・世界観・4キャラの詳細設計・8属性戦闘システム・12ヶ月イベントカレンダー・3エンディング分岐・UI設計方針
- **Unityプロジェクト構造の構築**: DESIGN.md準拠のディレクトリ構成
- **コアシステムC#スクリプト群（16ファイル）**:
  - `Core/`: GameManager, StateMachine, EventBus, PlayerStats, GameInitializer
  - `Time/`: TimeManager, CalendarData, TimeEnums
  - `Battle/`: BattleManager, EnemyData
  - `Social/`: ResonanceManager
  - `Data/`: CharacterData, ResonanceFormData, SkillData, PlayerStatsData, ItemData
  - `Dialogue/`: DialogueSystem
  - `UI/`: UIManager, TitleScreenUI, CalendarUI, DialogueUI, BattleUI
  - `Audio/`: AudioManager
  - `Field/`: FieldManager
  - `Save/`: SaveManager
- **会話データ（JSON）2本**: プロローグ初日バスシーン / 桜庭凛 共鳴ランク5「壁」シーン

### なぜそれを選んだか
- Day 0のプロジェクトにとって、聖典なき実装は方向を失う。まず設計書を確立することが最重要
- 次に、全機能の土台となるコアアーキテクチャがなければ、どの機能も作れない
- 会話データは「このゲームの感情的品質」の基準点として、最も重要なシーン（凛の壁）を先に書いた

### 実装してみての気づき・反省
- 属性を「感情」と対応させる設計（炎=怒り、氷=悲嘆…）は、世界観との一貫性が非常に高い。戦闘が単なるメカニクスにならない
- 共鳴ランクの戦闘ダメージ倍率連動は1.0〜1.5倍と控えめに設定。ペルソナの合体経験値ボーナスを参考に、体感で「仲が良いと強い」と感じるライン
- 凛のランク5台詞は書いていて胸が痛くなった。この品質を全キャラ維持できるかが最大の課題

## 設計メモ（意思決定の記録）

| 日付 | 決定 | 理由 |
|---|---|---|
| 2026-06-10 | タイトル「残響のアルカディア」に決定 | 「残響＝言えなかった言葉の余韻」「アルカディア＝偽りの楽園」がコアテーマと完全一致 |
| 2026-06-10 | エンジンはUnity 2022.3 LTS | ペルソナ風UIの再現にはUnityのUI制御の柔軟性が最適。UniTask+DOTweenの演出生産性 |
| 2026-06-10 | 属性を感情と対応させる | 炎=怒り、氷=悲嘆…とすることで戦闘が世界観の一部になる。「弱点を突く＝その感情を暴く」という暗喩 |
| 2026-06-10 | 共鳴ランク5に「壁」を設定 | ペルソナの社会リンクでは中盤に関係が深まる転換点がある。ランク5は「本当の傷を見せる」瞬間であり、プレイヤーに最も強い感情を与えるポイント |
| 2026-06-10 | 主人公の「言えなかった言葉」は最後に明かす | 全キャラの共鳴を通じて他者の沈黙を解いてきた主人公が、最後に自分自身と向き合う構造。これがトゥルーエンドの条件 |
| 2026-06-10 | 捨てた案: Unreal Engine | GASは強力だが、ペルソナ風の2DリッチUI表現にはUnityの方が適している |
